# Elasticsearch Query DSL Transactions Exercises

One-hour beginner practice on fake payment transactions indexed into `app-transactions-*`.

These exercises use the transaction data generated by `generate_fake_transactions.py`. The goal is to discover Elasticsearch Query DSL step by step using simple payment examples. Each exercise has the answer directly below it.

## Data Fields

The generated transactions use these fields:

| Field | Example | Use |
| --- | --- | --- |
| `@timestamp` | `2026-06-04T10:30:00Z` | Transaction time |
| `transaction_id` | `txn-a1b2c3d4e5f6a7b8` | Exact transaction lookup |
| `order_id` | `ord-123456` | Exact order lookup |
| `customer_id` | `cus-4821` | Exact customer filtering |
| `merchant_id` | `mer-204` | Exact merchant filtering |
| `payment_method` | `card`, `wallet`, `bank_transfer`, `paypal` | Payment method filtering |
| `card_network` | `visa`, `mastercard`, `amex`, `none` | Card network filtering |
| `currency` | `USD`, `EUR`, `MAD`, `GBP` | Currency filtering |
| `amount` | `247.95` | Numeric range and metrics |
| `status` | `authorized`, `captured`, `declined`, `failed`, `refunded`, `chargeback` | Transaction status |
| `status_code` | `200`, `201`, `402`, `409`, `500` | Numeric status filtering |
| `risk_score` | `0.837` | Numeric risk filtering |
| `latency_ms` | `1320` | Numeric latency filtering |
| `country` | `MA`, `FR`, `US`, `GB`, `ES` | Country filtering |
| `city` | `Casablanca`, `Paris`, `New York` | City filtering |
| `channel` | `web`, `mobile`, `pos`, `api` | Channel filtering |
| `provider` | `stripe`, `adyen`, `checkout-com`, `local-bank` | Payment provider filtering |
| `fraud_flag` | `true`, `false` | Boolean filtering |
| `error_reason` | `insufficient_funds`, `provider_error`, `customer_dispute` | Failure reason filtering |
| `message` | `Payment captured for order ord-123456` | Full-text search |

Use this index pattern:

```http
app-transactions-*
```

If you used the default generator on June 4, 2026, the concrete index name is:

```http
app-transactions-2026.06.04
```

The generator creates keyword mappings for exact fields, so queries can use fields like `status`, `payment_method`, and `provider` directly.

## Timing

| Time | Topic |
| --- | --- |
| 0-10 min | Mapping and first search |
| 10-25 min | `match`, `match_phrase`, `term` |
| 25-40 min | `terms`, `range`, `bool` |
| 40-55 min | Aggregations |
| 55-60 min | Final recap |

## 1. Inspect The Mapping

Before writing queries, check which fields exist and how Elasticsearch mapped them.

### Exercise

Inspect the mapping for `app-transactions-*`.

### Answer

```http
GET /app-transactions-*/_mapping
```

Optional compact view:

```http
GET /app-transactions-*/_field_caps?fields=@timestamp,transaction_id,order_id,payment_method,currency,amount,status,status_code,risk_score,latency_ms,country,channel,provider,fraud_flag,error_reason,message
```

## 2. First Search With `match_all`

`match_all` returns transactions without filtering.

### Exercise

Return transactions from `app-transactions-*`.

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "match_all": {}
  }
}
```

## 3. Full-Text Search With `match`

Use `match` for text fields such as `message`.

### Exercise

Find transactions where the message contains `failed`.

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "match": {
      "message": "failed"
    }
  }
}
```

## 4. Phrase Search With `match_phrase`

Use `match_phrase` when words must appear together in the same order.

### Exercise

Find transactions containing the phrase `Payment captured`.

### Answer

## 7. Multiple Exact Values With `terms`

Use `terms` when a field can match one value from a list.

### Exercise

Find transactions where `status` is `declined` or `failed`.

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "terms": {
      "status": [
        "declined",
        "failed"
      ]
    }
  }
}
```

## 10. Date Range

Use `range` on `@timestamp` to search a time window.

### Exercise

Find transactions from the last 15 minutes.

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "range": {
      "@timestamp": {
        "gte": "now-15m",
        "lte": "now"
      }
    }
  }
}
```

## 11. Boolean Filter

Use `bool` with `filter` when all conditions must be true.

### Exercise

Find transactions that are:

- paid by `card`
- status is `captured`
- amount is greater than or equal to `500`

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "bool": {
      "filter": [
        {
          "term": {
            "payment_method": "card"
          }
        },
        {
          "term": {
            "status": "captured"
          }
        },
        {
          "range": {
            "amount": {
              "gte": 500
            }
          }
        }
      ]
    }
  }
}
```

## 13. Combine Text Search And Filter

Use `must` for text search and `filter` for exact conditions.

### Exercise

Find card transactions where the message contains `declined`.

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "bool": {
      "must": [
        {
          "match": {
            "message": "declined"
          }
        }
      ],
      "filter": [
        {
          "term": {
            "payment_method": "card"
          }
        }
      ]
    }
  }
}
```

## 14. Count Transactions By Status

Aggregations summarize documents instead of focusing on individual hits.

### Exercise

Count transactions by `status`.

### Answer

```http
GET /app-transactions-*/_search
{
  "size": 0,
  "aggs": {
    "transactions_by_status": {
      "terms": {
        "field": "status",
        "size": 10
      }
    }
  }
}
```

## 16. Average Amount

Metric aggregations calculate numeric values.

### Exercise

Calculate the average transaction `amount`.

### Answer

```http
GET /app-transactions-*/_search
{
  "size": 0,
  "aggs": {
    "average_amount": {
      "avg": {
        "field": "amount"
      }
    }
  }
}
```

## 17. Average Amount By Currency

You can put a metric aggregation inside a bucket aggregation.

### Exercise

For each currency, calculate average transaction amount.

### Answer

```http
GET /app-transactions-*/_search
{
  "size": 0,
  "aggs": {
    "by_currency": {
      "terms": {
        "field": "currency",
        "size": 10
      },
      "aggs": {
        "average_amount": {
          "avg": {
            "field": "amount"
          }
        }
      }
    }
  }
}
```

## 18. Count Transactions Over Time

Use `date_histogram` to see how transactions are distributed over time.

### Exercise

Count transactions per hour.

### Answer

```http
GET /app-transactions-*/_search
{
  "size": 0,
  "aggs": {
    "transactions_per_hour": {
      "date_histogram": {
        "field": "@timestamp",
        "fixed_interval": "1h"
      }
    }
  }
}
```

## 19. Final Recap Query

### Exercise

Find important payment incidents:

- `status` is `declined`, `failed`, or `chargeback`
- `risk_score` is greater than or equal to `0.8`
- `amount` is greater than or equal to `500`
- event happened in the last day

### Answer

```http
GET /app-transactions-*/_search
{
  "query": {
    "bool": {
      "filter": [
        {
          "terms": {
            "status": [
              "declined",
              "failed",
              "chargeback"
            ]
          }
        },
        {
          "range": {
            "risk_score": {
              "gte": 0.8
            }
          }
        },
        {
          "range": {
            "amount": {
              "gte": 500
            }
          }
        },
        {
          "range": {
            "@timestamp": {
              "gte": "now-1d",
              "lte": "now"
            }
          }
        }
      ]
    }
  }
}
```
